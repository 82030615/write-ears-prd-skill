# 会员积分系统 PRD

## 1. 文档信息
- **版本**: v1.0
- **日期**: 2026-04-12
- **作者**: AI Assistant
- **状态**: 已批准

## 2. 背景与目标
构建会员积分系统，支持积分获取、积分兑换、积分过期、积分冻结等核心场景。需考虑会员等级差异（不同等级积分倍率不同）、活动期双倍积分、积分商城兑换库存不足、恶意刷积分等复杂情况。

### 6.2 Complex 句式使用规范验证

**规范要求**：组合条件不超过 3 个为宜

| 需求 ID | 条件数 | 是否符合规范 | 说明 |
|:---|:---:|:---:|:---|
| REQ-002 ~ REQ-006 | 2 | ✅ 符合 | 标准双条件组合 |
| REQ-007 | 3 | ✅ 符合 | 三条件组合，业务需要 |
| REQ-037, REQ-038 | 3 | ✅ 符合 | 三条件组合，业务需要 |
| 其他 Complex | 2 | ✅ 符合 | 标准双条件组合 |

### 6.3 Complex 句式 vs 简单句式对比

```
❌ 不推荐：拆分为多个简单需求
When 会员消费时，如果会员是 Silver 等级，系统给予 1.2 倍积分
When 会员消费时，如果会员是 Gold 等级，系统给予 1.5 倍积分
...

✅ 推荐：使用 Complex 句式
While 会员处于 Silver 等级，When 消费时，系统应当给予 1.2 倍积分
While 会员处于 Gold 等级，When 消费时，系统应当给予 1.5 倍积分
...

优势：
1. 明确表达"状态 + 事件"的组合关系
2. 符合 EARS 规范的标准句式
3. 便于生成覆盖度矩阵验证
```
