# Test #6: 模糊输入测试 - 极简输入

## 用户输入
> 订单系统

## 技能响应

您提到了"订单系统"，但这个描述比较简略，我需要更多信息才能为您编写完整的 EARS 格式 PRD。

### 请补充以下信息：

#### 1. 核心业务实体
- 系统涉及哪些核心业务对象？
  - 例如：订单 (Order)、用户 (User)、商品 (Product)、支付 (Payment)
  - 是否有库存 (Inventory)、物流 (Shipment)、优惠券 (Coupon) 等实体？

#### 2. 系统状态
- 订单有哪些关键状态？
  - 例如：待支付、已支付、已发货、已完成、已取消
  - 是否有退款状态、售后状态？

#### 3. 触发事件
- 有哪些用户操作或系统事件会触发业务逻辑？
  - 例如：用户下单、支付成功、用户取消、系统自动取消
  - 是否有发货确认、签收确认、退款申请等事件？

#### 4. 异常场景
- 需要处理哪些异常情况？
  - 例如：支付超时、库存不足、订单重复提交
  - 是否有网络异常、第三方支付失败、物流异常等情况？

#### 5. 约束规则
- 有哪些业务规则或限制条件？
  - 例如：订单支付时限（如下单后 30 分钟未支付自动取消）
  - 是否有库存锁定规则、退款时限规则？

#### 6. 非功能性需求
- 是否有性能、安全、可用性方面的要求？
  - 例如：并发处理能力、数据加密要求、系统可用性目标
