# 内容管理系统 PRD

## 1. 文档信息
- **版本**: v1.0
- **日期**: 2026-04-12
- **作者**: AI Assistant
- **状态**: 已批准

## 2. 背景与目标
构建内容管理系统，支持文章发布、审核、上线、下线流程，支持富文本编辑、图片上传、定时发布等功能。需妥善处理审核不通过、内容违规、定时任务失败等异常情况。

## 5. 功能性需求 (EARS Requirements)

### 5.1 文章创建与编辑模块

| Req ID | 模式 | 需求描述 |
|:---|:---|:---|
| REQ-001 | When | **When** 作者创建新文章时，系统应当生成草稿状态的文章记录 |
| REQ-002 | When | **When** 作者保存文章时，系统应当保存内容并记录保存时间 |
| REQ-003 | When | **When** 作者上传图片时，系统应当验证图片格式和大小 |
| REQ-004 | If | **If** 上传图片格式不符合要求，则系统应当提示"格式不支持" |
| REQ-005 | If | **If** 上传图片超过大小限制，则系统应当提示"文件过大" |
| REQ-006 | While | **While** 文章处于 Draft 状态期间，系统应当允许作者随时编辑 |
| REQ-007 | When | **When** 作者提交审核时，系统应当将文章状态更新为 PendingReview |

### 5.2 文章审核模块

| Req ID | 模式 | 需求描述 |
|:---|:---|:---|
| REQ-008 | When | **When** 文章进入审核队列时，系统应当通知审核员 |
| REQ-009 | When | **When** 审核员通过审核时，系统应当将文章状态更新为 Approved |
| REQ-010 | When | **When** 审核员拒绝审核时，系统应当将文章状态更新为 Rejected |
| REQ-011 | If | **If** 文章被标记为内容违规，则系统应当记录违规类型并通知作者 |
| REQ-012 | If | **If** 审核超时 2 小时未处理，则系统应当自动升级至高级审核员 |
| REQ-013 | While | **While** 文章处于 PendingReview 状态期间，作者不应当能修改内容 |
| REQ-014 | Complex | **While** 文章处于审核中状态下，**If** 作者申请撤回，则系统应当暂停审核并返回草稿状态 |

### 5.3 文章发布模块

| Req ID | 模式 | 需求描述 |
|:---|:---|:---|
| REQ-015 | When | **When** 审核通过的文章选择立即发布时，系统应当立即上线 |
| REQ-016 | When | **When** 作者设置定时发布时，系统应当在指定时间自动上线 |
| REQ-017 | When | **When** 文章成功发布时，系统应当将状态更新为 Published |
| REQ-018 | If | **If** 定时发布失败，则系统应当通知作者并保留待发布状态 |
| REQ-019 | If | **If** 发布时检测到内容违规，则系统应当阻止发布并标记异常 |
| REQ-020 | While | **While** 文章处于 Published 状态期间，系统应当对外展示内容 |

### 5.4 文章下线与删除模块

| Req ID | 模式 | 需求描述 |
|:---|:---|:---|
| REQ-021 | When | **When** 作者主动下线文章时，系统应当将状态更新为 Offline |
| REQ-022 | When | **When** 收到版权投诉时，系统应当立即下线相关文章 |
| REQ-023 | When | **When** 文章被标记为违规时，系统应当自动下线并通知作者 |
| REQ-024 | If | **If** 文章下线时正在被用户浏览，则系统应当保留缓存 5 分钟 |
| REQ-025 | While | **While** 文章处于 Offline 状态期间，系统不应当对外展示 |
| REQ-026 | Complex | **While** 文章处于 Published 状态下，**When** 收到严重违规举报时，系统应当立即下线并启动复核 |

### 5.5 内容违规处理模块

| Req ID | 模式 | 需求描述 |
|:---|:---|:---|
| REQ-027 | When | **When** 内容被检测为违规时，系统应当标记违规类型 |
| REQ-028 | When | **When** 违规内容被确认时，系统应当扣除作者信用分 |
| REQ-029 | If | **If** 作者累计违规达到 3 次，则系统应当暂停其发布权限 7 天 |
| REQ-030 | If | **If** 作者累计违规达到 5 次，则系统应当永久封禁账号 |
| REQ-031 | If | **If** 违规申诉通过，则系统应当恢复文章状态并返还信用分 |
| REQ-032 | While | **While** 作者处于发布限制状态期间，系统应当阻止新文章提交 |

### 5.6 定时发布异常处理模块

| Req ID | 模式 | 需求描述 |
|:---|:---|:---|
| REQ-033 | If | **If** 定时任务执行失败，则系统应当记录失败原因 |
| REQ-034 | If | **If** 定时发布时文章状态不是 Approved，则系统应当取消发布 |
| REQ-035 | If | **If** 定时发布时作者账号被限制，则系统应当取消发布 |
| REQ-036 | When | **When** 定时发布失败后，系统应当通知作者重新设置 |
| REQ-037 | Complex | **While** 系统处于维护期间，**If** 定时发布任务触发，则系统应当推迟至维护结束后执行 |
